# Формат файлового контейнера КриптоПро — результаты анализа

Заметки по устройству файлового контейнера, полученные при разработке на живом
стенде (КриптоПро CSP 5.0.13000, Рутокен Lite/ECP). Это наш собственный анализ
поведения провайдера через CryptoAPI и файловую систему носителя, а не
воспроизведение чьего-либо кода.

## Контейнер = шесть файлов

Файловый контейнер (флоппи-профиль) — это папка с шестью файлами:

| Файл | Назначение |
|---|---|
| `primary.key` | часть секретного ключа |
| `masks.key` | маска ключа + имитовставка (ГОСТ 28147) |
| `header.key` | открытые метаданные + встроенный сертификат |
| `primary2.key` | вторая часть |
| `masks2.key` | вторая маска |
| `name.key` | дружественное имя контейнера |

Один и тот же набор из шести файлов хранится:

- **на токене** — как файлы в папке файловой системы носителя (доступ стережёт
  чип через ACL);
- **в реестре** — как бинарные значения под `...\Crypto Pro\Settings\USERS\
  <SID>\Keys\<имя>\`;
- **на диске (HDIMAGE)** — папка `<имя>.000` в `%LOCALAPPDATA%\Crypto Pro`;
- **на сменном носителе (FAT12)** — папка в корне; имя ограничено форматом 8.3.

Ключевое следствие для резервного копирования: **приватный ключ в контейнере
зашифрован**, копируется как есть, без расшифровки. Поэтому копия не требует
экспортируемости ключа — переносится «чемодан», а не его содержимое.

## Носитель ↔ тип ключа

- **Файловый контейнер** (Рутокен Lite, FKC) — шесть файлов есть, копируется.
- **Аппаратный ключ** (крипта внутри чипа Рутокен ЭЦП) — извлекаемого блоба
  нет, копировать нечего. Определяется по отсутствию 6-файловой структуры на
  носителе; исключается из копирования заранее.

## Признак неэкспортируемости

Флаг экспортируемости приватного ключа лежит в `header.key`: **байт 25, бит
0x80** (ASN.1 BIT STRING в структуре метаданных). Проверено экспорт-тестом.

**Но на CSP 5.0.13000 просто переключить бит нельзя:** последние 4 байта
`header.key` — бесключевая контрольная сумма тела заголовка. После правки бита
без пересчёта суммы контейнер помечается как «header corrupted», и доступ к
ключу через API ломается. Сумма проприетарная (не CRC32 — проверено полным
перебором параметров; не тривиальное усечение ГОСТ-хэша). На части сборок CSP
контроля заголовка нет — там переключение проходит. См. дорожную карту.

## Связка CryptoAPI ↔ файловая система носителя

Уникальное имя контейнера из `PP_UNIQUE_CONTAINER` несёт мостик к папке на
токене: например `SCARD\rutoken_lt_39445e29\0A00\28A9` — сегмент `0A00` в
шестнадцатеричном виде (0x0A00 = 2560) совпадает с ID папки-контейнера в
файловой системе носителя. Это позволяет связать контейнер, найденный через
CryptoAPI, с его файлами, читаемыми напрямую с носителя.
