# Запись на токен и копирование токен → токен (проектная заметка)

Статус: обсуждение, не реализовано. Резюме анализа.

## Ключевое различие: «тупое» хранилище vs «умный» токен

| | Папка / реестр / HDIMAGE | Токен (Рутокен) |
|---|---|---|
| Как хранит | просто 6 файлов | файлы + on-token ACL, доступ стережёт чип |
| Запись | сырой `WriteFile` — работает | сырой записи мало: нужны правильные ACL |

Чтение с токена сырьём (rtComLite `ReadBinary`) безопасно — ACL не трогаем.
Запись сырьём небезопасна: без правильных ACL на `primary.key`/`masks.key`
либо чип отвергнет контейнер, либо приватный ключ станет доступен без PIN.
CryptoPro при создании контейнера выставляет эти ACL, аллоцирует ID папки,
привязывает к структуре носителя. Повторить сырьём = реверсить логику
создания контейнера Рутокена.

## Почему CryptoAPI-импорт не спасёт

`CryptImportKey(PRIVATEKEYBLOB)` требует, чтобы ключ был экспортируемым, а весь
парк неэкспортируемый. Инъекция ключа через API отпадает.

## Рекомендуемый путь — keycopy

`keycopy` копирует зашифрованный блоб контейнера как есть, CryptoPro сам
создаёт контейнер на приёмнике со всеми ACL.

- **Токен → токен:** `csptest -keycopy -contsrc "\\.\Reader A\<guid>"
  -contdest "\\.\Reader B\<новое-имя>"` + PIN обеих сторон (дефолт 12345678).
- **Архив → токен:** источник должен быть контейнером, который CryptoPro читает.
  HDIMAGE/реестр — читаются, keycopy напрямую. Архив `cert\ИНН.ММГГ\8hex` —
  сначала подложить в HDIMAGE-хранилище (умеем писать сыро), потом keycopy.

Цена: `keycopy` — это `csptest.exe` (CLI). Нам нужен только код возврата (не
парсинг вывода), а он стабилен; зависимость от CryptoPro при записи на токен
неустранима. Для примитива записи CLI приемлем.

## Механика, о которой помнить

1. PIN обеих сторон (дефолт подставляем сами, иначе спрашиваем; журнал по ТЗ 6).
2. Свободная память приёмника (`FreeMemory` через rtComLite) — Рутокен Lite
   64K тесный.
3. Уникальное имя контейнера на приёмнике; дружественное имя из `name.key`.
4. Аппаратные контейнеры (крипта в чипе) записать так же нельзя — блоба нет.
5. Проверка после записи: резолвер + `csptest -check`.

## Что проверить на стенде до реализации

- Точный синтаксис `keycopy` токен→токен и флаги PIN.
- Работает ли keycopy токен→токен одной командой или нужен промежуточный шаг.
- Поведение с дефолтным PIN без GUI (`-silent`?).
