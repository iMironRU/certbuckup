# Handoff: Диспетчер сертификатов (консольная утилита копирования сертификатов с токенов)

## Overview
Консольная (TUI-style) утилита для операторов, которые работают с криптографическими токенами (Rutoken, JaCarta, eToken и т.п.). Приложение показывает список подключённых к компьютеру токенов, список сертификатов на выбранном токене, и позволяет скопировать выбранный сертификат (с закрытым ключом) в одно из нескольких мест назначения — с учётом ограничений экспорта ключа. Интерфейс полностью управляется с клавиатуры (стрелки, Enter, Esc, F3/F5/F10), в стиле классических DOS/консольных менеджеров (двойная рамка окна, футер с F-клавишами, подсветка выбранной строки), но перекрашен в палитру дизайн-системы Nocturne (тёмный фон, один акцент), а не в классические синие цвета DOS-приложений.

## About the Design Files
Файлы в этом пакете — **дизайн-референс, сделанный в HTML** (Design Component/прототип), а не production-код. Это интерактивный кликабельный/клавиатурный прототип, демонстрирующий экраны, переходы и поведение — не то, что нужно копировать 1:1 в реальное приложение. Задача разработчика — **воссоздать это поведение и раскладку в целевой среде** (Windows Forms/WPF/Avalonia/консольный TUI-фреймворк типа Spectre.Console/Terminal.Gui — в зависимости от того, на чём фактически пишется утилита), используя её собственные паттерны, а не веб-стек.

## Fidelity
**High-fidelity (hifi).** Цвета, отступы, типографика и переходы между экранами показаны с намерением "как должно выглядеть в финале" — воспроизводить нужно точно (в рамках возможностей целевого UI-тулкита; см. раздел "Design Tokens" для точных значений).

## Screens / Views

Прототип — это один экран-«окно» (терминал) с несколькими режимами (screen state) и модальными диалогами поверх. Ниже — все состояния.

### 1. Tokens — список токенов
**Назначение:** стартовый экран; показывает все подключённые к компьютеру токены/смарт-карты.
**Layout:** одна колонка строк-устройств внутри окна, каждая строка — grid из 3 колонок: иконка (30px) · название+модель+метаданные (1fr) · тег состояния (auto).
**Компоненты:**
- Заголовок окна (breadcrumb): "Токены" — верхняя строка окна, слева.
- Индикатор количества устройств справа от breadcrumb: точка-индикатор (7×7px, круг, цвет акцента) + текст "устройств подключено: N".
- Список токенов, на каждую строку:
  - Иконка токена (USB-флешка/чип, не замок — см. Assets)
  - Название токена (напр. "Rutoken ЭЦП 2.0"), рядом приглушённым текстом модель ("— Rutoken ECP 2.0 flash")
  - Строка метаданных мельче: "Считыватель: USB Reader 0 · S/N 0A3F-77C1 · сертификатов: 2"
  - Если токен доступен только для чтения (нельзя копировать с него сертификаты): тег "только чтение" справа (стиль `.tag-outline`)
  - Выбранная строка (курсор) — заливка фоном `--color-accent-700`, текст `--color-accent-100`
- Пример данных (4 токена): Rutoken ЭЦП 2.0 (2 серт., экспорт разрешён), JaCarta PKI (1 серт., **экспорт с устройства запрещён**), Rutoken Lite (2 серт., экспорт разрешён), Аладдин eToken 5110 (0 серт.)

### 2. Certificates — список сертификатов на выбранном токене
**Назначение:** показывает сертификаты на токене, выбранном на предыдущем экране; отсюда инициируется копирование.
**Layout:** тот же паттерн строк, что и Tokens: grid 26px/1fr/auto.
**Компоненты:**
- Breadcrumb: "Токены › <имя токена>"
- На строку сертификата:
  - Иконка: документ/сертификат — если ключ можно копировать; иконка навесного замка — если ключ **нельзя** копировать (или потому что у сертификата стоит признак "неэкспортируемый", или потому что сам токен доступен только для чтения)
  - Владелец сертификата (ФИО или название организации) — основная строка
  - Строка метаданных: "<Издатель (УЦ)> · Действителен: <дата с>–<дата по> · Отпечаток: <8 символов hex>"
  - Если заблокирован: тег справа — "ключ неэкспортируемый" (если ограничение только на уровне сертификата) либо "устройство: только чтение" (если ограничение на уровне токена)
- Пустое состояние, если у токена нет сертификатов: центрированный приглушённый текст "На этом носителе не найдено сертификатов."

### 3. Диалог "Куда скопировать сертификат" (модальный, вызывается F5)
**Назначение:** выбор места назначения копирования.
**Layout:** модальное окно (`.dialog` из дизайн-системы) поверх затемнённого фона (`.dialog-backdrop`).
**Компоненты:**
- Заголовок: "Куда скопировать сертификат"
- Подзаголовок: имя выбранного сертификата (владелец)
- Список из 4 вариантов назначения, каждый — строка с радио-индикатором (кружок 14px, обводка акцентом, заливка акцентом если выбран) + название + путь мельче приглушённым:
  1. **Диск D:\** — `D:\Сертификаты\`
  2. **Реестр Windows** — `HKCU\...\Certificates`
  3. **Папка рядом с приложением** — `.\certs\`
  4. **Папка КриптоПро** — `%ProgramData%\Crypto Pro\Store\`
- **Чекбокс "Снять признак запрета на копирование закрытого ключа"** — появляется в этом же диалоге, под списком назначений, **только если у выбранного сертификата стоит признак неэкспортируемости**. По умолчанию не отмечен. Клик или пробел переключают.
- Подсказка внизу диалога: "↑↓ выбор · Enter подтвердить · Esc отмена"
- **Важно:** клик по строке назначения не просто выделяет её — он сразу подтверждает выбор и продолжает флоу (эквивалент выбора курсором + Enter).

### 4. Диалог "Копирование невозможно" (модальный)
**Назначение:** показывается вместо диалога выбора назначения, если сам токен запрещает экспорт (напр. JaCarta PKI).
**Компоненты:** заголовок "Копирование невозможно", текст: "Токен «<имя>» не позволяет копировать сертификаты и ключи на внешние носители. Экспорт заблокирован на аппаратном уровне.", кнопка OK (Enter/Esc закрывает).

### 5. Диалог "Информация" (F3, на экранах Tokens и Certificates)
**Назначение:** показать подробности о выбранном токене или сертификате, без возможности редактирования.
**Компоненты:** заголовок = имя токена/владельца сертификата; список пар ключ-значение:
- Для токена: Модель, Считыватель, Серийный №, Экспорт с устройства (разрешён/запрещён), Сертификатов
- Для сертификата: УЦ, Действителен с, Действителен по, Отпечаток, Алгоритм, Ключ экспортируемый (да/нет)
- Кнопка "Закрыть" (Esc)

### 6. Progress — экран копирования
**Назначение:** визуализирует процесс копирования после подтверждения.
**Компоненты:** центрированный блок: текст "Копирование сертификата «<владелец>» → <назначение>…", горизонтальный прогресс-бар (420×14px, трек `--color-neutral-900`, заполнение `--color-accent-500`, анимируется от 0 до 100% рывками ~120мс), процент под баром.
**Поведение:** авто-переход на экран Result по завершении (~1.5–2 сек).

### 7. Result — результат операции
**Назначение:** показывает итог последнего копирования + журнал всех операций за сеанс.
**Компоненты:**
- Статусная плашка (фон `--color-neutral-900`, рамка `--color-divider`): иконка (галочка в круге при успехе, крестик в круге при ошибке) + сообщение о результате + строка метаданных (владелец · токен · назначение)
- Таблица "Журнал операций сеанса" (`.table` из дизайн-системы): колонки Время / Токен / Владелец / Назначение / Статус (тег "Успешно" зелёного/акцентного тона или "Ошибка" нейтрального тона — см. Design Tokens, в системе нет отдельного цвета ошибки)
- Сообщения об ошибке при отклонённом запрете: "Экспорт закрытого ключа запрещён (CRYPT_E_NOT_EXPORTABLE). Копирование отменено."
- Сообщение при снятом признаке: "Признак запрета снят, сертификат и ключ скопированы успешно."

### 8. Диалог "Завершить сеанс?" (F10)
**Назначение:** подтверждение выхода. Два варианта ("Да, завершить сеанс" / "Нет, остаться"), клик по варианту сразу выполняет действие. При подтверждении — сброс к экрану Tokens и очистка журнала операций за сеанс.

## Interactions & Behavior

**Клавиатурная схема (глобальный обработчик keydown):**
- `↑` / `↓` — перемещение курсора по текущему списку (токены / сертификаты / назначения — внутри диалога) с циклическим переходом (после последнего — на первый)
- `Enter` — открыть выбранный токен (на экране Tokens) / подтвердить выбор в диалоге / вернуться к списку сертификатов (на экране Result)
- `Esc` / `Backspace` — назад (Certificates → Tokens, Result → Certificates), либо закрыть диалог
- `F3` — открыть диалог "Информация" о выбранном элементе
- `F5` — начать копирование выбранного сертификата (открывает диалог назначения или диалог блокировки)
- `F10` — открыть диалог подтверждения выхода
- `Пробел` — переключить чекбокс "снять признак запрета" внутри диалога назначения
- Все действия также доступны кликом мыши (по строке списка, по F-клавише в футере, по опции диалога)

**Переходы (state machine):**
```
Tokens --Enter/клик--> Certificates
Certificates --Esc/Backspace--> Tokens
Certificates --F5 (токен read-only)--> [диалог: Копирование невозможно] --Esc/Enter--> Certificates
Certificates --F5 (токен разрешает)--> [диалог: Куда скопировать]
  диалог: выбор назначения (+ чекбокс, если сертификат неэкспортируемый)
  --Enter/клик по назначению-->
    если сертификат экспортируемый ИЛИ чекбокс отмечен --> Progress --(авто, ~1.5-2с)--> Result (успех)
    если сертификат неэкспортируемый И чекбокс не отмечен --> Progress --> Result (ошибка)
Result --Enter/Esc--> Certificates
любой экран --F10--> [диалог: Завершить сеанс?] --Да--> Tokens (сброс) / --Нет--> назад
```

**Анимации/переходы:** прогресс-бар — плавное изменение `width` (CSS transition ~120ms linear) на каждый тик таймера (~120мс, случайный шаг 8-18%). Мигающий курсор-декорация (▮) в заголовке окна — blink каждую 1с (чисто декоративный элемент, не функциональный).

**Валидация:** F5/F3 недоступны (визуально приглушены, opacity 0.4), если на токене нет сертификатов.

## State Management
Ключевое состояние экрана (single state machine):
- `screen`: `tokens | certs | progress | result`
- `modal`: `null | dest | blocked | info | exit`
- `tokenCursor`, `certCursor`, `destCursor` — индексы курсора в соответствующих списках
- `selectedTokenIdx` — какой токен открыт
- `destFlagClear` — состояние чекбокса "снять запрет" в диалоге назначения (сбрасывается в `false` при каждом открытии диалога назначения)
- `infoTarget` — `{type: 'token'|'cert', idx}` для диалога информации
- `exitCursor` — курсор в диалоге подтверждения выхода
- `progressPct` — 0–100, драйвит прогресс-бар
- `lastResult` — `{ts, owner, tokenName, dest, status: 'ok'|'error', message}` последней операции
- `log` — массив всех `lastResult` за сеанс, самая новая запись сверху

**Реальные данные, которые нужно будет подключить (в прототипе — статичный мок):**
- Список подключённых токенов/считывателей (обычно через PKCS#11 / CryptoAPI CSP)
- Список сертификатов на каждом токене: владелец, издатель (УЦ), срок действия, отпечаток (thumbprint), алгоритм, признак экспортируемости закрытого ключа
- Флаг "устройство доступно только для чтения" — определяется по возможностям конкретного токена/CSP
- Реальная операция копирования сертификата + закрытого ключа в одно из 4 назначений, включая снятие признака `CRYPT_EXPORTABLE`, если пользователь это подтвердил (требует прав/API соответствующего CSP — операция может быть необратима или требовать пароль/PIN токена, это не отражено в прототипе и стоит уточнить у бизнеса)

## Design Tokens
Из дизайн-системы **Nocturne** (`design_system/styles.css`, `design_system/nocturne_readme.md`):
- Фон: `--color-bg` `#161826`; поверхности/окно: `--color-surface` `#232532`
- Текст: `--color-text` `#e9e9ed`
- Акцент (единственный): `--color-accent` `#9184d9`, тональная рампа `--color-accent-100…900`
- Нейтральная рампа: `--color-neutral-100…900`
- Радиусы: `--radius-sm` 4px, `--radius-md` 8px, `--radius-lg` 14px
- Отступы: `--space-1…8` (2.8px…22.4px, компактная плотность 0.7×)
- Тени: `--shadow-sm/md/lg`
- Шрифт интерфейса дизайн-системы: Inter (`--font-heading`, `--font-body`)
- **Отступление от системы:** сам "терминал" (весь текст внутри окна утилиты) использует **моноширинный шрифт** (`'JetBrains Mono', 'SFMono-Regular', Consolas, 'Liberation Mono', monospace`) вместо Inter — это осознанное решение ради консольной эстетики TUI; Inter из дизайн-системы не используется внутри этого экрана
- Рамка окна: `3px double var(--color-accent-600)` — двойная линия, отсылка к классическим TUI-рамкам (Norton Commander/FAR-стиль), но в акцентном цвете системы, а не в стандартном синем
- В системе нет отдельного семантического цвета "ошибка/успех" — статус "Успешно" использует `.tag-accent`, статус "Ошибка" — `.tag-neutral` (различие через форму иконки: галочка в круге / крестик в круге, а не через цвет)

## Assets
Иконки нарисованы вручную инлайн-SVG в стиле Phosphor (line icons, stroke 1.6px, скруглённые концы) — **не импортированы из реальной библиотеки Phosphor**, разработчику стоит заменить их на настоящие иконки Phosphor (или аналог) при реализации:
- Токен/устройство: иконка USB-накопителя (корпус + разъём)
- Сертификат (доступен для копирования): иконка документа с уголком и линиями текста
- Заблокированный ключ/сертификат: иконка навесного замка
- Успех: галочка в круге
- Ошибка: крестик в круге
- Чекбокс "снять запрет": галочка в квадрате при отмеченном состоянии

Фото/изображения не используются — экран полностью текстовый/иконочный, соответствует консольной эстетике задачи.

## Files
- `Диспетчер сертификатов TUI.dc.html` — единственный файл прототипа (шаблон + логика состояния). Открывается напрямую в браузере.
- `design_system/styles.css` — токены и классы дизайн-системы Nocturne, на которые опирается прототип (`.tag`, `.dialog`, `.table`, `.hr`, переменные `--color-*`, `--space-*`, `--radius-*`, `--shadow-*`)
- `design_system/nocturne_readme.md` — полное описание дизайн-системы Nocturne (принципы, цвета, типографика, компоненты)
